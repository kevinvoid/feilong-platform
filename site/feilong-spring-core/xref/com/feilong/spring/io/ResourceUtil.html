35  /**
36   * {@link org.springframework.core.io.Resource} 工具类.
37   * 
38   * <h3>Resource接口:</h3>
39   * 
40   * <blockquote>
41   * <p>
42   * 
43   * <ul>
44   * <li>{@link Resource#getInputStream()} 定位并打开资源,返回读取此资源的一个 InputStream.<br>
45   * <b>每次调用预期会返回一个新的 InputStream,由调用者负责关闭这个流.</b></li>
46   * <li>{@link Resource#exists()}返回标识这个资源在物理上是否的确存在的 boolean 值.</li>
47   * <li>{@link Resource#isOpen()}返回标识这个资源是否有已打开流的处理类的 boolean 值.<br>
48   * 如果为 true,则此InputStream 就不能被多次读取, 而且只能被读取一次然后关闭以避免资源泄漏. <br>
49   * 除了 InputStreamResource,常见的resource实现都会返回 false.</li>
50   * <li>{@link Resource#getDescription()}返回资源的描述,一般在与此资源相关的错误输出时使用. 此描述通常是完整的文件名或实际的URL地址.</li>
51   * </ul>
52   * 
53   * </p>
54   * </blockquote>
55   * 
56   * 
57   * <h3>Spring提供了很多 Resource 的实现:</h3>
58   * 
59   * <blockquote>
60   * <p>
61   * 
62   * <blockquote>
63   * <table border="1" cellspacing="0" cellpadding="4" summary="">
64   * <tr style="background-color:#ccccff">
65   * <th align="left">实现</th>
66   * <th align="left">说明</th>
67   * </tr>
68   * <tr valign="top">
69   * <td>{@link UrlResource}</td>
70   * <td>封装了java.net.URL,它能够被用来访问任何通过URL可以获得的对象,例如:文件、HTTP对象、FTP对象等.<br>
71   *   所有的URL都有个标准的 String表示,这些标准前缀可以标识不同的URL类型, 包括 file:访问文件系统路径, http: 通过HTTP协议访问的资源, ftp: 通过FTP访问的资源等等.<br>
72   * {@link UrlResource} 对象可以在Java代码中显式地使用 UrlResource 构造函数来创建. 但更多的是通过调用带表示路径的 String 参数的API函数隐式地创建.<br>
73   * 
74   * 在后一种情况下,JavaBeans的 {@link java.beans.PropertyEditor} 会最终决定哪种类型的 Resource 被创建.<br>
75   * 如果这个字符串包含一些众所周知的前缀,比如 classpath:,它就会创建一个对应的已串行化的 Resource. <br>
76   * 然而,如果不能分辨出这个前缀,就会假定它是个标准的URL字符串,然后创建{@link UrlResource}.</td>
77   * </tr>
78   * <tr valign="top" style="background-color:#eeeeff">
79   * <td>{@link ClassPathResource}</td>
80   * <td>这个类标识从classpath获得的资源. 它会使用线程context的类加载器(class loader)、给定的类加载器或者用来载入资源的给定类. <br>
81   * 如果类路径上的资源存在于文件系统里,这个 Resource 的实现会提供类似于java.io.File的功能. <br>
82   * 而如果资源是存在于还未解开(被servlet引擎或其它的环境解开)的jar包中,这些 Resource 实现会提供类似于java.net.URL 的功能.   {@link ClassPathResource}
83   * 对象可以在Java代码中显式地使用ClassPathResource 构造函数来创建.<br>
84   * 但更多的是通过调用带表示路径的String参数的API函数隐式地创建.<br>
85   * 在后一种情况下,JavaBeans的 PropertyEditor 会分辨字符串中 classpath: 前缀,然后相应创建 {@link ClassPathResource}.</td>
86   * </tr>
87   * <tr valign="top" >
88   * <td>{@link FileSystemResource}</td>
89   * <td>这是为处理 java.io.File 而准备的Resource实现. <br>
90   * 它既可以作为File提供,也可以作为URL.</td>
91   * </tr>
92   * <tr valign="top" style="background-color:#eeeeff">
93   * <td>{@link "org.springframework.web.context.support.ServletContextResource"}</td>
94   * <td>这是为 ServletContext 资源提供的 Resource 实现,它负责解析相关web应用根目录中的相对路径. <br>
95   * 它始终支持以流和URL的方式访问. 但是只有当web应用包被解开并且资源在文件系统的物理路径上时,才允许以 java.io.File 方式访问. <br>
96   * 是否解开并且在文件系统中访问,还是直接从JAR包访问或以其它方式访问如DB(这是可以想象的),仅取决于Servlet容器.</td>
97   * </tr>
98   * <tr valign="top" >
99   * <td>{@link InputStreamResource}</td>
100  * <td>这是为给定的 InputStream 而准备的 Resource 实现. 它只有在没有其它合适的 Resource 实现时才使用.<br>
101  * 而且,只要有可能就尽量使用{@link ByteArrayResource} 或者其它基于文件的Resource 实现. <br>
102  * 与其它 Resource 实现不同的是,这是个 已经 打开资源的描述符-因此 isOpen() 函数返回 true.<br>
103  * 如果你需要在其它位置保持这个资源的描述符或者多次读取一个流,请不要使用它.</td>
104  * </tr>
105  * <tr valign="top" style="background-color:#eeeeff">
106  * <td>{@link ByteArrayResource}</td>
107  * <td>这是为给定的byte数组准备的 Resource 实现. 它会为给定的byte数组构造一个 {@link ByteArrayResource}. <br>
108  * 它在从任何给定的byte数组读取内容时很有用,因为不用转换成单一作用的 InputStreamResource.</td>
109  * </tr>
110  * </table>
111  * </blockquote>
112  * 
113  * </p>
114  * </blockquote>
115  *
116  * @author <a href="http://feitianbenyue.iteye.com/">feilong</a>
117  * @see org.springframework.core.io.Resource
118  * @see org.springframework.core.io.ResourceLoader
119  * @see org.springframework.core.io.UrlResource
120  * @see org.springframework.core.io.ClassPathResource
121  * @see org.springframework.util.ResourceUtils
122  * @see "org.springframework.web.context.support.ServletContextResource"
123  * @since 1.0.9
124  */
125 public final class ResourceUtil{
126 
127     /** The Constant log. */
128     private static final Logger LOGGER = LoggerFactory.getLogger(ResourceUtil.class);
129 
130     //---------------------------------------------------------------
131 
132     /** Don't let anyone instantiate this class. */
133     private ResourceUtil(){
134         //AssertionError不是必须的. 但它可以避免不小心在类的内部调用构造器. 保证该类在任何情况下都不会被实例化.
135         //see 《Effective Java》 2nd
136         throw new AssertionError("No " + getClass().getName() + " instances for you!");
137     }
138 
139     //---------------------------------------------------------------
140 
141     /**
142      * 获得 resource.
143      *
144      * @param urlOrPath
145      *            the url or path
146      * @return the resource
147      * @see org.springframework.core.io.DefaultResourceLoader#DefaultResourceLoader()
148      * @see org.springframework.core.io.ResourceLoader
149      */
150     public static Resource getResource(String urlOrPath){
151         ResourceLoader resourceLoader = new DefaultResourceLoader();
152         return resourceLoader.getResource(urlOrPath);
153     }
154 
155     //---------------------------------------------------------------
156 
157     /**
158      * Resolve the given resource location to a {@code java.net.URL}.
159      * <p>
160      * Does not check whether the URL actually exists; simply returns the URL that the given location would correspond to.
161      * </p>
162      * 
163      * @param resourceLocation
164      *            the resource location to resolve: either a "classpath:" pseudo URL, a "file:" URL, or a plain file path
165      * @return a corresponding URL object
166      * @see org.springframework.util.ResourceUtils#getURL(String)
167      * @since 1.4.1
168      */
169     public static URL toURL(String resourceLocation){
170         try{
171             return ResourceUtils.getURL(resourceLocation);
172         }catch (FileNotFoundException e){
173             LOGGER.error("", e);
174             throw new UncheckedIOException("resourceLocation:[" + resourceLocation + "] not found!!!", e);
175         }
176     }
177 }
